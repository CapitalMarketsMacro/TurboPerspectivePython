# AGENTS.md — FX Executions Blotter (Angular + Perspective)

## Architecture Overview

Single-page Angular 21 standalone app: `AppComponent` → `BlotterComponent` → `<perspective-viewer>` web component.
Data flows over a **WebSocket** from a separate Python Perspective server (port 8080) — there is no REST API or Angular `HttpClient`.

```
Python Perspective Server :8080/websocket
        ↓ WebSocket (perspective.websocket)
BlotterComponent (src/app/blotter/)
        ↓ viewer.load(client)
<perspective-viewer> web component
```

## Perspective Loading Pattern (Critical)

Perspective JS/WASM assets are **vendored into `public/perspective/`** — they are intentionally excluded from Angular's build pipeline to avoid WASM bundler complexity.

Loading sequence in `blotter.ts`:
1. `loadPerspective()` dynamically injects `<script type="module" src="perspective/init.js">` at runtime.
2. `public/perspective/init.js` imports all Perspective modules, sets `window.__perspective`, and fires a `perspective-ready` CustomEvent.
3. `BlotterComponent.ngOnInit()` awaits that event, then calls `(window as any).__perspective.websocket(wsUrl)`.

**Never** import `@perspective-dev/*` packages directly in Angular component code — use `window.__perspective` after `perspective-ready`.

## Custom Elements

Both `App` and `BlotterComponent` declare `schemas: [CUSTOM_ELEMENTS_SCHEMA]` because `<perspective-viewer>` is a custom web component unknown to the Angular compiler. Any new component hosting a Perspective viewer must include this schema.

## Viewer Configuration

The viewer is configured via `viewer.restore()` in `BlotterComponent.ngOnInit()`. Key defaults:

- **Table**: `fx_executions` (name must match the Python server's registered table)
- **Plugin**: `Datagrid`
- **Theme**: `Pro Dark` (default)
- **Columns** (15): `trade_id`, `exec_time`, `ccy_pair`, `side`, `notional`, `rate`, `spot_rate`, `pnl`, `venue`, `trader_id`, `desk`, `status`, `counterparty`, `value_date`, `sequence_num`
- **Sort**: `exec_time` descending

To add/remove columns or change layout, edit the `restore()` call in `blotter.ts`.

## Themes

All 10 themes are loaded as `<link>` tags in `src/index.html` (not Angular imports) and registered via `viewer.resetThemes([...])`. To add a new theme: add the CSS file to `public/perspective/`, link it in `index.html`, and add its name to the `resetThemes` array.

## Layouts & Styling

- `src/styles.scss`: global reset — `html/body` are `100%` height, `overflow: hidden`, dark background `#0f0f1a`.
- `blotter.scss` uses `:host { display: flex }` to make the viewer fill the viewport. Maintain full-bleed flex layout when adding new views.
- SCSS is the project style language; Prettier uses `printWidth: 100`, `singleQuote: true`.

## Developer Workflows

```bash
npm start          # dev server at http://localhost:4200 (requires Python backend on :8080)
npm run build      # production build → dist/
npm run watch      # dev build with watch
ng test            # unit tests via Vitest (no test files generated by default — skipTests: true)
```

`ng generate component <name>` creates `.ts`, `.html`, `.scss` — no spec file (all schematics set `skipTests: true` in `angular.json`).

## Adding a New Blotter/View

1. Create a new standalone component under `src/app/<name>/`.
2. Add `CUSTOM_ELEMENTS_SCHEMA` and replicate the `loadPerspective()` / `viewer.restore()` pattern from `blotter.ts`.
3. Import the component in `app.ts` and add its selector to `app.html`.
4. If connecting to a different table, change the `table:` key in `restore()`.

## Key Files

| File | Purpose |
|---|---|
| `src/app/blotter/blotter.ts` | Core Perspective integration pattern |
| `public/perspective/init.js` | Boots all Perspective modules, fires `perspective-ready` |

| `angular.json` | `skipTests: true` for all schematics; `@angular/build:application` builder |
| `public/perspective/` | Vendored Perspective JS/WASM assets (do not delete or re-bundle) |
